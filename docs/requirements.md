# Requirements

## 機能要件

### ファイル指定

- 比較するファイルを左右それぞれ以下で指定できる
  - オーナー / リポジトリ名（例: `octocat/Hello-World`）
  - ブランチ名 / タグ名 / コミット SHA（例: `main`, `v1.0.0`, `abc1234`）
  - ファイルパス（例: `src/index.ts`）
- 各項目はフォームで入力する
- Owner / Repository フィールドに GitHub のファイル URL（`https://github.com/{owner}/{repo}/blob/{ref}/{path}`）を貼り付けると全フィールドを自動補完できる
- Ref フィールドはブランチ・タグ一覧からの補完候補を表示する（Owner / Repository 入力後、フォーカスアウト時に取得）
- File Path フィールドはリポジトリのファイルツリーから補完候補を表示する（Ref 入力後、フォーカスアウト時に取得）
- 左右すべてのフィールド（owner / repo / ref / path）が入力済みの場合のみ比較ボタンを有効化する

### 認証

- トークン未入力の場合、パブリックリポジトリのみ比較可能
- GitHub PAT を入力した場合、プライベートリポジトリも比較可能
- 使用する PAT は Classic PAT（スコープ: `repo`）とする
- PAT はブラウザの `localStorage` に保存し、次回以降の入力を省略できる
- PAT をクリアするボタンを提供する
- プライベートリポジトリにアクセスしようとしてトークンがない場合、案内メッセージを表示する

### 差分表示

- サイドバイサイド（左右並列）表示
- ユニファイド（unified diff）表示
- 表示モードを切り替えるトグルを提供する
- 追加行・削除行・変更行をそれぞれ色分け表示する

### その他

- ファイル取得中はローディング表示を行う
- エラー（リポジトリ不在・ファイル不在・認証エラーなど）は Left / Right それぞれに対応した位置に個別表示する
- 比較状態は URL クエリパラメータに反映し、URL 共有を可能にする

## 非機能要件

- GitHub Pages に静的サイトとしてデプロイできること（`next export` または `output: 'export'`）
- GitHub API レート制限（未認証: 60 req/時）に対して適切なエラーメッセージを表示すること
- モバイルブラウザでも基本的に閲覧できること（レスポンシブ対応）
